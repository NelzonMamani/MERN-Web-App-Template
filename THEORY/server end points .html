List of Endpoints for a Normal Backend Server (server.js)

  1. GET /users - Retrieve a list of all users

  2. GET /users/:id - Retrieve a specific user by id

  3. POST /users - Create a new user

  4. PATCH /users/:id - Update a specific user by id

  5. DELETE /users/:id - Delete a specific user by id

  6. GET /items - Retrieve a list of all items

  7. GET /items/:id - Retrieve a specific item by id

  8. POST /items - Create a new item

  9. PATCH /items/:id - Update a specific item by id

  10. DELETE /items/:id - Delete a specific item by id

  11. GET /orders - Retrieve a list of all orders

  12. GET /orders/:id - Retrieve a specific order by id

  13. POST /orders - Create a new order

  14. PATCH /orders/:id - Update a specific order by id

  15. DELETE /orders/:id - Delete a specific order by id

  16. GET /reviews - Retrieve a list of all reviews

  17. GET /reviews/:id - Retrieve a specific review by id

  18. POST /reviews - Create a new review

  19. PATCH /reviews/:id - Update a specific review by id

  20. DELETE /reviews/:id - Delete a specific review by id

  21. GET /categories - Retrieve a list of all categories

  22. GET /categories/:id - Retrieve a specific category by id

  23. POST /categories - Create a new category

  24. PATCH /categories/:id - Update a specific category by id

  25. DELETE /categories/:id - Delete a specific category by id

  26. GET /cart - Retrieve a user's cart

  27. POST /cart - Add an item to a user's cart

  28. DELETE /cart/:id - Remove an item from a user's cart

  29. GET /checkout - Retrieve order details for checkout

  30. POST /checkout - Submit order for checkout

  31. GET /wishlist - Retrieve a user's wishlist

  32. POST /wishlist - Add an item to a user's wishlist

  33. DELETE /wishlist/:id - Remove an item from a user's wishlist

  34. GET /search - Search for items by keyword

  35. GET /filters - Retrieve filters for item search

  36. GET /statistics - Retrieve various statistics for the server.

  37. GET /logs - Retrieve various logs for the server.

  38. GET /backup - Retrieve backup data for the server.

  39. POST /backup - Create a backup data for the server.

  40. PATCH /backup - Update a backup data for the server.

  41. DELETE /backup - Delete a backup data for the server.

In a typical authentication and authorization setup, the authServer.js would handle user registration and authentication related tasks such as login, logout, and token issuance.

The main server, server.js, would handle the CRUD operations for all models and other functionality such as handling file uploads and sending emails.
